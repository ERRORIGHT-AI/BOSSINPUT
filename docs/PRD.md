# BOSSINPUT 产品需求文档 (PRD)

**版本**: 2.0
**日期**: 2025-02-21
**项目**: BOSSINPUT - 语音输入小键盘软件

---

## 1. 产品概述

### 1.1 产品定位

**BOSSINPUT** 是一款面向极客开发者的客制化小键盘软件伴侣。它结合了硬件按键配置与本地语音转文字功能，让开发者能够通过物理按键快速触发语音输入，实现高效的「vibe coding」工作流。

### 1.2 核心价值主张

| 价值点 | 描述 |
|-------|------|
| **本地优先，隐私安全** | 所有语音处理在本地 CPU 完成，数据不上云 |
| **极简工作流** | 一键触发语音输入，文字自动粘贴到当前光标位置 |
| **开发者友好** | 针对 AI 编程工具、终端、编辑器深度优化 |
| **跨平台支持** | macOS、Windows、Linux 三平台统一体验 |

### 1.3 目标用户

主要用户是技术极客和开发者，特别是：
- 使用 AI 编程助手（Claude Code、Cursor 等）
- 频繁在终端工作
- 需要快速录入代码注释和文档
- 关注数据隐私，倾向本地化解决方案

### 1.4 用户痛点分析

| 痛点 | 描述 | 影响 |
|-----|------|-----|
| 上下文切换频繁 | 开发者需要在多个终端/编辑器/面板间快速切换 | 中断心流 |
| 语音输入隐私顾虑 | 现有方案需上传云端，代码/敏感信息有泄露风险 | 企业/个人开发者担忧 |
| 配置分散 | 键盘配置和语音输入需要两个应用 | 体验割裂 |
| 通用方案不适配 | 通用语音输入不支持开发者场景（代码语法、命令行） | 效率低下 |

### 1.5 市场机会

- **AI 编程工具普及**：Claude Code、Cursor 等工具兴起，带来新的交互需求
- **本地 STT 成熟**：Parakeet、Whisper 等模型在 CPU 上已可用
- **客制化键盘增长**：Vial/QMK 社区活跃，开发者用户基数扩大

## 2. 核心功能

### 2.1 键盘配置管理

基于 Vial 协议实现小键盘的按键配置功能：

- **按键映射编辑器**：可视化键盘布局，拖拽方式配置每个按键的功能
- **旋钮配置**：
  - **旋转动作**：配置左旋/右旋触发的事件
  - **按压动作**：配置按下触发的事件
- **图层管理**：支持多图层切换，通过专用键在图层间切换
- **宏命令支持**：录制和编辑按键宏序列
- **配置导入/导出**：支持配置文件备份和分享
- **实时应用**：配置修改即时生效，无需重新编译固件

### 2.2 智能旋钮 - 多终端切换

**核心功能**：旋钮自动适配当前激活应用，实现无缝的多终端/面板切换：

| 当前应用 | 旋钮左旋 | 旋钮右旋 | 旋钮按压 |
|---------|---------|---------|---------|
| VS Code | 切换到上一个面板/编辑器 | 切换到下一个面板/编辑器 | 切换文件浏览器焦点 |
| macOS Terminal | 切换到上一个终端窗口 | 切换到下一个终端窗口 | 新建终端窗口 |
| IntelliJ IDEA | 切换到上一个工具窗口 | 切换到下一个工具窗口 | 聚焦到编辑器 |
| Chrome/Firefox | 切换到上一个标签页 | 切换到下一个标签页 | 新建标签页 |
| Claude Code | 切换到上一个会话/面板 | 切换到下一个会话/面板 | 新建对话 |

**实现方式**：
- 应用监听当前激活窗口的进程名称
- 根据预设的应用规则表，动态映射旋钮动作
- 支持用户自定义应用规则和快捷键映射

### 2.3 语音转录功能

基于 Handy 的 STT 引擎实现语音转文字：

- **开关模式触发**：按一次语音键开始录音，再按一次停止并转录
- **CPU 优先模型**：默认使用 Parakeet V3（CPU 优化），中端 CPU 上达 5x 实时速度
- **多语言支持**：自动语言检测，支持中英文混合输入
- **智能粘贴**：转录结果自动粘贴到当前光标位置
- **静音检测**：VAD 自动过滤静音片段

### 2.4 应用集成

针对开发者场景的深度集成：

- **跨应用粘贴**：在终端、编辑器、浏览器等任何应用中工作
- **焦点感知**：智能识别当前激活窗口，确保文字输入到正确位置
- **CLI 参数支持**：可被其他程序通过命令行参数调用

### 2.5 MVP 功能范围

#### P0 - MVP 必须功能 (macOS)

## 3. 技术架构

### 3.1 整体架构选择

采用 **Tauri (Rust + React/TypeScript)** 作为主应用框架：

- **跨平台一致性**：一套代码编译到 macOS/Windows/Linux
- **轻量高效**：Rust 后端提供出色的性能和内存安全
- **Handy 代码复用**：Handy 本身基于 Tauri，可直接复用其 STT 模块
- **前端生态成熟**：React 生态丰富，适合构建复杂 UI

### 3.2 后端架构 (Rust)

```
┌─────────────────────────────────────────────────────────┐
│                    Tauri Backend                         │
├─────────────────┬─────────────────┬─────────────────────┤
│  Keyboard Module │  Audio Module   │   Transcription      │
│  (Vial Protocol) │  (CPAL + VAD)   │   (Parakeet/Whisper) │
├─────────────────┴─────────────────┴─────────────────────┤
│            Device I/O & Text Injection                   │
│  (HID USB / rdev / enigo / clipboard)                   │
└─────────────────────────────────────────────────────────┘
```

**核心模块**：

| 模块 | 功能 | 核心依赖 |
|-----|------|---------|
| Keyboard Module | 移植 Vial 的 HID 通信协议，实现按键配置读写 | hidapi |
| Audio Module | 音频采集、静音检测 | cpal, vad-rs |
| Transcription Module | 语音转文字，支持 Parakeet V3 | transcription-rs, whisper-rs |
| Device I/O | 全局事件监听、文字注入 | rdev, enigo |

### 3.3 前端架构 (React + TypeScript)

```
┌─────────────────────────────────────────────────────────┐
│                     React UI                             │
├─────────────────┬─────────────────┬─────────────────────┤
│  Keyboard Config │  Voice Settings │   Model Manager     │
│  (布局编辑器)     │  (触发模式等)    │   (模型下载/切换)    │
├─────────────────┴─────────────────┴─────────────────────┤
│              Tauri IPC Bridge                            │
└─────────────────────────────────────────────────────────┘
```

## 5. 数据流与交互逻辑

### 5.1 语音转录流程

```
用户按下语音键
      │
      ▼
┌─────────────────────────────────────────────────────────┐
│  1. 应用检测按键事件 (rdev 全局钩子)                      │
│     - 验证是否来自已配对的 BOSSINPUT 键盘                │
│     - 切换录音状态（开始/停止）                           │
└─────────────────────────────────────────────────────────┘
      │
      ▼
┌─────────────────────────────────────────────────────────┐
│  2. 录音中状态                                           │
│     - 显示浮动反馈窗口                                    │
│     - 启动音频采集 (cpal)                                 │
│     - VAD 实时检测语音活动                                │
│     - 托盘图标闪烁提示                                    │
└─────────────────────────────────────────────────────────┘
      │
      ▼ (用户再次按下语音键)
┌─────────────────────────────────────────────────────────┐
│  3. 停止录音 & 转录                                      │
│     - 停止音频采集                                        │
│     - 静音裁剪（去除首尾静音）                            │
│     - 调用 Parakeet V3 模型进行转录                       │
│     - 更新状态为"转录中..."                               │
└─────────────────────────────────────────────────────────┘
      │
      ▼
┌─────────────────────────────────────────────────────────┐
│  4. 文字输出                                             │
│     - 获取当前光标位置窗口                                │
│     - 通过剪贴板或直接输入方式粘贴文字                    │
│     - 隐藏反馈窗口                                        │
│     - 可选：显示转录结果预览（几秒后自动消失）            │
└─────────────────────────────────────────────────────────┘
```

### 5.2 旋钮切换流程

```
用户旋转/按压旋钮

      ▼
┌─────────────────────────────────────────────────────────┐
│  1. 应用检测事件 (rdev + HID 通信)                       │

│  2. 应用识别                                             │
│     - 获取当前激活窗口的进程名称/Bundle ID                │
│     - 匹配预设的应用规则表                                │
│     - 如果无匹配，使用默认行为（如标签页切换）            │

│  3. 动作执行                                             │
│     - 根据规则表查找对应的快捷键                          │
│     - 发送快捷键组合到系统 (enigo/rdev)                   │
│     - 视觉/触觉反馈（可选）                               │
└─────────────────────────────────────────────────────────┘
```

## 7. 错误处理策略

### 7.1 错误分类

| 错误类型 | 示例 | 用户提示 | 恢复方式 |
|---------|------|---------|---------|
| 设备未连接 | 键盘断开 | "未检测到键盘，请检查连接" | 自动重连/手动重试按钮 |
| 权限不足 | 麦克风被拒绝 | "需要麦克风权限才能录音" | 打开系统设置引导 |
| 模型缺失 | 模型文件损坏 | "语音模型不可用，请重新下载" | 一键重新下载 |
| 转录失败 | 音频质量问题 | "无法识别语音，请重试" | 提示用户靠近麦克风 |
| 网络错误 | 模型下载失败 | "下载失败，请检查网络连接" | 重试/稍后重试 |

### 7.2 错误提示原则

- **清晰具体**：告诉用户发生了什么
- **可执行**：提供解决方案或操作按钮
- **不中断**：非致命错误不阻塞主要流程
- **日志记录**：所有错误记录到本地日志供调试

## 8. 发布与分发

### 8.1 MVP 发布计划

**阶段 1: Alpha (内部测试)**
- 目标：核心功能可用
- 范围：macOS，键盘配置 + 基础语音转录
- 分发方式：直接分享 .app 文件

**阶段 2: Beta (小范围测试)**
- 目标：稳定性验证，收集反馈
- 范围：添加模型管理、错误处理
- 分发方式：GitHub Releases (测试版)

**阶段 3: v1.0 (公开发布)**
- 目标：正式面向开发者社区
- 范围：完整的 MVP 功能
- 分发方式：GitHub Releases + Homebrew Cask

### 8.2 分发渠道

| 平台 | 渠道 | 状态 |
|-----|------|-----|
| macOS | GitHub Releases | MVP |
| macOS | Homebrew Cask | v1.0 |

### 8.3 版本策略

- **语义化版本**：遵循 SemVer (Major.Minor.Patch)

## 11. 非功能性需求

### 11.1 兼容性要求

**操作系统**：
- macOS 11+ (Intel & Apple Silicon) — **MVP 首选平台**
- Windows 10/11 (x64) — v1.1+
- Ubuntu 20.04+ / 其他主流发行版 — v1.1+

**硬件要求**：
- CPU: 支持 SSE4.2 的现代处理器
- RAM: 8GB+
- 磁盘: 4GB+ 可用空间（为不同模型预留）

**外设**：
- BOSSINPUT 客制化小键盘（Vial 协议兼容）

### 11.2 可用性要求

- **隐私优先**：所有数据处理在本地完成，不上传任何音频或文字
- **离线可用**：首次下载模型后，完全离线运行
- **状态可见**：托盘图标和反馈窗口清晰显示当前状态
- **容错处理**：键盘断开、录音失败等异常情况有明确提示
- **配置迁移**：支持配置导入导出，方便跨设备迁移
- **模型扩展性**：架构支持未来接入其他 STT 模型

### 11.3 安全性要求

- 最小权限原则：仅请求必需的系统权限（麦克风、USB 设备访问）
- 本地数据加密：配置文件敏感信息加密存储
- 代码安全：遵循 Rust 安全最佳实践
